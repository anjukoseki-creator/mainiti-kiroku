# 毎日記録 — 設計ドキュメント

> 「昨日の自分を超えるためのAI成長ログ」
> 比較対象は他人ではなく、過去の自分。

---

## 1. 要件整理

### プロダクトの本質
- 日記でも感情の記録でもない。**事実（できた・学んだ・次やる）だけを30秒で残す**アプリ。
- 主役はAI。ログの積み重ねから「自分でも気付いていない成長」を発見する。
- 商用ではなく毎日自分が使うもの。**拡張性より継続体験を最優先**。

### 機能要件（MVP）

| # | 機能 | 内容 |
|---|------|------|
| F1 | ホーム | 今日の日付 / 継続日数 / 今日の記録状況 / 前回AIレビューからの日数 |
| F2 | 今日の記録 | できたこと最大3件・学んだこと1〜3行・明日最重要1件 → 保存 |
| F3 | カレンダー | 記録日に色。クリックで内容表示 |
| F4 | 過去ログ | 一覧・検索・日付順 |
| F5 | AIレビュー | 伸びていること / 努力の傾向 / 改善ポイント / 過去との比較 / 来週へのアドバイス |

### 非機能要件
- 完全ローカル動作。DBなし、`localStorage` で永続化。
- 記録は**30秒以内・最小クリック**で完了。
- 将来のデータ書き出し（JSONエクスポート/インポート）を最初から設計に含める。
- UIは「シンプル・静か・落ち着く・余白多め」。派手な演出なし。

### 技術スタック（承認済み）
- **React 18 + Vite + TypeScript**（PWA対応 → スマホのホーム画面にも置ける）
- 状態管理:React Context + カスタムフック（小規模なのでライブラリ不要）
- スタイル:CSS Modules + CSSカスタムプロパティ（依存を増やさない）
- AIレビュー:**Claude API（キー設定時は自動）/ プロンプトコピー（キー未設定時）** の両対応

## 2. 改善案の提案

要件に加えて、「毎日使い続ける」ために以下を提案します。

### 採用推奨（MVPに含める）
1. **ホーム＝入力画面にする**
   ホームと記録画面を分けず、開いた瞬間に今日の入力欄が出る。未記録なら入力欄、記録済みなら今日の内容＋編集ボタン。**クリック0回で入力開始**できる。
2. **「昨日の"明日やること"」を今日の画面に表示**
   昨日書いた「明日最も重要なこと」を今日のできたこと欄の上に表示し、ワンタップで「できたこと」に昇格できる。宣言→実行のループが閉じ、このアプリ最大の差別化になる。
3. **Enterで次の欄へ**
   できたこと1件目でEnter → 2件目、3件目 → 学んだこと…とキーボードだけで完結。30秒記録の要。
4. **継続日数は「猶予付き」**
   1日抜けただけでストリークが0になると開くのが嫌になる。「直近30日で◯日記録」も併記し、完璧主義で挫折しない設計にする。
5. **AIレビュー結果も保存・履歴化**
   レビュー自体が成長ログになる。「3か月前のレビューと今」を比較できる。

### 将来候補（MVP後）
- タグの自動抽出（「Python30分」→ #Python、努力の傾向グラフに使う）
- 週次サマリーの自動生成
- エクスポートの定期リマインド

## 3. 画面構成

ナビゲーションは下部（またはサイド）に4つだけ。

```
┌─────────────────────────────┐
│  ホーム   カレンダー   ログ   レビュー  │
└─────────────────────────────┘
```

### 3.1 ホーム（＝今日の記録）
```
  2026年7月12日（日）
  🔥 継続 14日 ・ 直近30日で 22日記録
  前回AIレビューから 6日

  ─ 昨日の宣言 ───────────────
  「基本情報の過去問を1年分解く」   [できた ✓]

  ─ 今日できたこと（最大3件）──────
  [ Python 30分            ]
  [ +                       ]

  ─ 今日学んだこと ────────────
  [                          ]

  ─ 明日最も重要なこと ──────────
  [                          ]

           [ 保存 ]
```
- 記録済みの日は読み取り表示＋「編集」。
- 保存は1ボタン。途中保存も可（空欄OK）。

### 3.2 カレンダー
- 月表示。記録日は淡い色のドット。今日は枠強調。
- 日をクリック → その日の記録をモーダル/下部パネルで表示。

### 3.3 過去ログ
- 新しい順のカードリスト（日付・できたこと・学んだこと・宣言）。
- 上部に検索ボックス（全文部分一致）。

### 3.4 AIレビュー
- 「レビューを実行」ボタン（対象期間:前回レビュー以降 or 直近30日）。
- APIキー設定済み → 自動で分析結果を表示・保存。
- 未設定 → 分析プロンプトを生成して「コピー」ボタン。結果を貼り付けて保存できる欄も用意。
- 過去のレビュー履歴を下に一覧表示。

### 3.5 設定（歯車アイコン）
- Claude APIキー（localStorageのみ、外部送信はAnthropic APIへの分析リクエストのみ）
- JSONエクスポート / インポート

### UIトーン
- 背景:オフホワイト、文字:濃グレー、アクセント:深緑1色のみ
- 余白広め、角丸大きめ、影は最小限、アニメーションはフェードのみ
- フォント:システムフォント（Noto Sans JP相当）

## 4. データ構造

`localStorage` にキーを分けて保存。すべてJSONで、エクスポートは全キーを1ファイルにまとめる。

```ts
// storage key: "mk:entries"  — 日付をキーにしたマップ
type ISODate = string; // "2026-07-12"

interface Entry {
  date: ISODate;
  achievements: string[];   // 今日できたこと（最大3）
  learning: string;         // 今日学んだこと
  tomorrowFocus: string;    // 明日最も重要なこと
  focusDone?: boolean;      // 昨日の宣言を達成したか
  createdAt: string;        // ISO datetime
  updatedAt: string;
}
type EntryMap = Record<ISODate, Entry>;

// storage key: "mk:reviews"
interface Review {
  id: string;               // crypto.randomUUID()
  createdAt: string;
  periodFrom: ISODate;
  periodTo: ISODate;
  source: "api" | "manual"; // API自動 or 貼り付け
  content: string;          // Markdown
}

// storage key: "mk:settings"
interface Settings {
  apiKey?: string;
  model: string;            // 既定: claude-sonnet-5
}

// エクスポート形式（将来の移行にも使う）
interface ExportFile {
  app: "mainichi-kiroku";
  version: 1;
  exportedAt: string;
  entries: EntryMap;
  reviews: Review[];
}
```

- ストレージアクセスは `storage.ts` の1モジュールに集約。将来 IndexedDB やファイル保存に替えるときもここだけ差し替えればよい。
